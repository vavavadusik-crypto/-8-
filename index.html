Браузерная ИИ-модель (через локальный мост, без API-ключа) исследует тему и собирает карточки прямо на доску. Дальше — обычный рендер.
Свой или бесплатный API (Ollama, OpenRouter, Hugging Face…). Ключ уходит на локальный worker только на время запроса и не попадает в проект, localStorage и manifest.

Настоящий локальный MP4

Работает только через локальный `npm run dev`: Board ставит job в приватный FFmpeg worker, проверяет MP4/WAV и возвращает manifest. Публикация не выполняется.
Без ключа: ИИ рисует фон под каждую сцену. Выключено — тёмный брендированный фон. Свой FAL-ключ (в настройках) даёт премиум-качество.
Ключи живут только в памяти локального worker до его перезапуска и не попадают в проект, localStorage и manifest.
